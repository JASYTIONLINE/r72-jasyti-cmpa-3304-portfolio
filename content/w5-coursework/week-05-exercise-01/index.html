Interactive Buttons

This CSS exercise explores the difference between :hover and :active pseudo-classes. Each button demonstrates how user interface feedback is handled when hovering over or clicking on a button.

  • The Hover: state responds to the pointer entering the button area, giving visual feedback before the user clicks.
  • The Active state applies during the actual mouse click, simulating a pressed-in appearance.

Both are used to create intuitive and satisfying interactions.

Button Style 01: Color Fade

This button smoothly transitions its background and text colors on hover using background-color and color properties. On click, it retains the same appearance but the effect remains to emphasize simplicity and elegance.

Button Style 02: Lift and Shadow

This button rises slightly and adds a soft drop shadow on hover using transform: translateY and box-shadow. On click, the lift is reduced and the shadow softens to simulate the button being pressed down.

Button Style 03: Slide Effect

This button slides slightly to the right on hover using transform: translateX. On click, the slide distance is reduced to give the appearance of the button being pressed inward.

Button Style 04: Glow Border

This button adds a glowing border and switches text color on hover using box-shadow and color. On click, the glow dims slightly and the text becomes darker, simulating a visual press effect.

Button Style 05: Scale and Shadow

This button enlarges and casts a bold shadow on hover using transform: scale and box-shadow. On click, the button shrinks slightly and the shadow is softened, simulating a realistic click press.

Button Style 06: Invert Colors

This button inverts its background and text color on hover using background-color and color. On click, it uses a lighter background and deeper text color to represent a button being clicked.
