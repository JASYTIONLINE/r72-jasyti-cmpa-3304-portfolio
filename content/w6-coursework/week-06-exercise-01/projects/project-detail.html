Project Detail Page

This page demonstrates an advanced navigation architecture that combines multiple navigation patterns on a single page. As a detail page nested within the Projects section, it showcases how different navigation patterns can work together to provide both global site navigation and section-specific navigation.

Two-Column Layout Architecture

This page implements a two-column flexbox layout that demonstrates responsive design principles:

  • Sidebar Column (60px fixed width): Contains the vertical sidebar navigation, providing persistent access to main site sections
  • Content Column (flexible width): Houses the page content and section-specific navigation, adapting to available space

This layout pattern is ideal for detail pages where users need both site-wide navigation and section-specific navigation options.

Dual Navigation System

This page exemplifies a dual navigation approach, combining two distinct navigation patterns:

Global Navigation: Sidebar Pattern

The left sidebar provides persistent global navigation to all main category pages (Home, About, Projects, Gallery). This fixed vertical navigation remains visible during page scroll, ensuring users can always navigate to other major sections of the site. The sidebar uses the vertical list navigation pattern (Navigation Pattern 01), which is ideal for secondary navigation that doesn't compete with primary content.

Section Navigation: Custom Centered Pattern

The custom centered navigation bar at the top of the content area provides section-specific navigation within the Projects category. This navigation pattern (Navigation Pattern 03) demonstrates:

  • Centered, contained design: The navigation bar is visually distinct with a light gradient background and rounded corners, creating a clear separation from page content
  • Scalable architecture: The flexbox layout with justify-content: space-evenly automatically distributes navigation items evenly, accommodating additional links as the section grows
  • Visual hierarchy: Larger font size (1.5rem) and prominent styling indicate these are the most likely next navigation actions within the current section
  • Contextual navigation: Links within this bar navigate to related pages in the same section (e.g., "Return to Projects Home"), providing intuitive breadcrumb-style navigation

Relative Path Navigation

This nested page demonstrates proper relative path usage for multi-page sites:

  • ../index.html - Navigates up one directory level to reach the homepage
  • ../about.html - Goes up one level to access root-level pages
  • index.html - Stays within the current directory (projects folder) to access the Projects landing page
  • ../gallery/ - Goes up one level, then into a different folder to access the Gallery section

Understanding relative paths is essential for building maintainable multi-page websites where pages are organized into logical folder structures.

CSS Architecture: Shared Stylesheet

This page references the shared stylesheet using ../css/style.css, demonstrating how nested pages access CSS files located at the project root. This approach ensures:

  • Consistent styling across all pages
  • Single source of truth for design changes
  • Efficient file management without CSS duplication

All navigation patterns, layout utilities, and styling are defined in the shared stylesheet, making site-wide design updates simple and consistent.
